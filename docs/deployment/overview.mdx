Idun Agent Platform supports two deployment modes depending on your needs. You can run a single agent with a config file, or deploy the full platform with centralized management.

## Deployment modes

<Tabs>
  <Tab title="Production (pre-built images)">
    Pull published Docker images from Docker Hub and start the full stack. No build step required — this is the recommended approach for production and self-hosted deployments.

    ```bash
    git clone https://github.com/Idun-Group/idun-agent-platform.git
    cd idun-agent-platform
    # Edit docker-compose.yml: set AUTH__SECRET_KEY to a strong random value
    docker compose up -d
    ```

    The Manager API is available at `http://localhost:8000` and the web UI at `http://localhost:3000`.

    This mode suits:
    - Production deployments
    - Self-hosted environments
    - Multi-agent deployments
    - Teams that need centralized configuration and RBAC
  </Tab>
  <Tab title="Development (hot reload)">
    Build images locally from source with hot reload enabled for the Manager and web UI. Requires the full repository checkout.

    ```bash
    git clone https://github.com/Idun-Group/idun-agent-platform.git
    cd idun-agent-platform
    cp .env.example .env
    docker compose -f docker-compose.dev.yml up --build
    ```

    This mode suits:
    - Local development and testing
    - Contributing to the platform
  </Tab>
  <Tab title="Standalone engine">
    Run a single agent from a YAML config file. No Manager, no database, no web UI required.

    ```bash
    pip install idun-agent-engine
    idun agent serve --source file --path config.yaml
    ```

    This mode suits:
    - Single-agent deployments
    - CI/CD pipelines where you version config in Git
  </Tab>
</Tabs>

## Agent lifecycle management

<Tabs>
  <Tab title="Manager UI">
    From the agent detail page, manage the agent lifecycle:

    - The status badge shows **Draft** or **Active**
    - Click **Restart** to call `/reload` on the running engine, which re-fetches config from the platform
    - The Overview tab includes **Connect to Platform** instructions for enrollment

    ![Agent detail overview tab](/images/ui/agents-detail-overview.png)
  </Tab>
  <Tab title="Config file">
    With file-based deployment, manage your agent by restarting the process or updating the config file and restarting:

    ```bash
    # Stop the running agent
    pkill -f "idun agent serve"

    # Update config.yaml, then restart
    idun agent serve --source file --path config.yaml
    ```

    For managed deployments, restart the agent container to force a fresh config fetch from the Manager.
  </Tab>
</Tabs>

## Full stack architecture

In managed mode, the platform consists of three services and a database:

| Component | Description | Default port |
|---|---|---|
| **Manager** | FastAPI control plane API for agent configuration, auth, and workspace management | 8000 |
| **Web UI** | React admin dashboard | 3000 |
| **PostgreSQL** | Stores agent configs, user accounts, workspace data | 5432 |
| **Agent engines** | Your agent services, each running the Idun engine SDK | 8008+ |

```yaml docker-compose.yml (excerpt)
services:
  db:
    image: postgres:16
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: idun_agents
    ports:
      - "5432:5432"

  manager:
    image: freezaa9/idun-ai:latest
    ports:
      - "8000:8000"
    environment:
      DATABASE__URL: postgresql+asyncpg://postgres:postgres@db:5432/idun_agents
      AUTH__SECRET_KEY: your-super-secret-key-at-least-32-characters-long
      PORT: 8000
    depends_on:
      db:
        condition: service_healthy

  web:
    image: freezaa9/idun-ai-web:latest
    ports:
      # nginx inside the container listens on 8080
      - "3000:8080"
    environment:
      - API_URL=http://localhost:8000
    depends_on:
      - manager
```

## Prerequisites

| Requirement | Standalone | Managed |
|---|---|---|
| Python 3.12+ | Yes | Yes |
| Docker + Docker Compose | No | Yes |
| PostgreSQL 16 | No | Yes (included in Docker Compose) |
| `idun-agent-engine` package | Yes | Yes (for agent services) |
