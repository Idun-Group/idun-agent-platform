Google ADK (Agent Development Kit) is Google's framework for building Gemini-powered agents. Idun Agent Platform wraps ADK agents as production services with AG-UI streaming, guardrails, and observability.

<Tip>
  Want to start from working code? The [agent templates](/templates) include ADK examples for tool calling and structured I/O.
</Tip>

## Create an ADK agent

<Tabs>
  <Tab title="Manager UI">
    Create the agent in the Manager, then connect your code.

    <Steps>
      <Step title="Open the agent wizard">
        From the [Agent Dashboard](/manager/overview), click **Create an agent**. In step 1, enter a name and select **Google ADK**.

        ![Agent creation wizard with ADK selected](/images/ui/agents-create-step1.png)
      </Step>
      <Step title="Configure the framework">
        In step 2, fill in:

        - **Agent definition**: Python module path and variable, e.g. `my_agent.agent:root_agent`
        - **App name**: application name for ADK session/memory scoping
        - **Agent Host**: Localhost or Remote
        - **Server Port**: the port your agent will listen on

        The agent definition uses Python module path notation (`module.path:variable`), not a file path. This differs from LangGraph.
      </Step>
      <Step title="Enroll your agent">
        Step 3 gives you the enrollment instructions. In your agent's project directory:

        ```bash
        pip install idun-agent-engine google-adk
        export IDUN_MANAGER_HOST=http://localhost:8000
        export IDUN_AGENT_API_KEY=<your-api-key>
        idun agent serve --source manager
        ```

        Get the API key from the **API Integration** tab on your agent's detail page.
      </Step>
      <Step title="Add session and memory backends">
        Back in the Manager, open your agent's **Overview** tab and click **Manage** next to Memory. ADK supports:

        - **Session services**: In-memory, Database (PostgreSQL), Vertex AI
        - **Memory services**: In-memory, Vertex AI

        The defaults are in-memory for both. Switch to Database or Vertex AI for production persistence.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Config file">
    Set `agent.type` to `ADK` and provide the agent definition path and `app_name`.

    ```yaml config.yaml
    server:
      api:
        port: 8008

    agent:
      type: ADK
      config:
        name: "my-adk-agent"
        agent: "my_agent.agent:root_agent"
        app_name: "myagent"
    ```

    Then run:

    ```bash
    pip install idun-agent-engine google-adk
    idun agent serve --source file --path config.yaml
    ```
  </Tab>
</Tabs>

## Write your agent code

Create an ADK agent at the module path matching your configuration:

```python my_agent/agent.py
from google.adk.agents import Agent

root_agent = Agent(
    model="gemini-2.5-flash",
    name="weather_agent",
    description="An agent that answers questions about the weather.",
    instruction="You are a helpful weather assistant. Answer questions about weather conditions.",
)
```

<Note>
  If you use Vertex AI models, authenticate with `gcloud auth application-default login` before running the agent.
</Note>

### The `agent` field

The value uses Python module path notation with a colon separator:

- **Module path**: dotted Python module path (e.g., `my_agent.agent`)
- **Variable name**: the agent instance in that module (e.g., `root_agent`)

This differs from LangGraph's `graph_definition`, which uses a file path.

## Session services

Session services control how ADK persists session state. When using the Manager UI, configure this from the Memory section. With a config file, set the `session_service` field.

<Tabs>
  <Tab title="In-memory (default)">
    Sessions are held in process memory. Lost on restart. This is the default.

    ```yaml
    session_service:
      type: in_memory
    ```
  </Tab>
  <Tab title="Database">
    Sessions persist to PostgreSQL or another SQLAlchemy-compatible database.

    ```yaml
    session_service:
      type: database
      db_url: "postgresql://user:password@localhost:5432/sessions"
    ```
  </Tab>
  <Tab title="Vertex AI">
    Sessions are managed by Google Vertex AI Reasoning Engine.

    ```yaml
    session_service:
      type: vertex_ai
      project_id: "my-gcp-project"
      location: "us-central1"
      reasoning_engine_app_name: "my-reasoning-engine-app"
    ```
  </Tab>
</Tabs>

## Memory services

Memory services control long-term memory persistence, separate from session state.

<Tabs>
  <Tab title="In-memory (default)">
    Memory is held in process memory. This is the default.

    ```yaml
    memory_service:
      type: in_memory
    ```
  </Tab>
  <Tab title="Vertex AI">
    Memory is managed by Google Vertex AI Memory Bank.

    ```yaml
    memory_service:
      type: vertex_ai
      project_id: "my-gcp-project"
      location: "us-central1"
      memory_bank_id: "my-memory-bank-id"
    ```
  </Tab>
</Tabs>

<Warning>
  ADK does not support folder paths that contain spaces. Make sure your project directory path has no spaces.
</Warning>
