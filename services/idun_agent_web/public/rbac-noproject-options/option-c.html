Copy tradeoffs: The banner copy is deliberately brief — one line for the situation, one inline CTA. The skeleton grid carries the page's spatial meaning without any copy at all. For the non-owner path the banner text gives the exact instruction ("Settings › Projects") without asking the user to figure out where to go. Risk: two-line banners can be skipped by users who pattern-match them as notices they've already read.

Affordance tradeoffs: Highest spatial continuity of the three options — the page doesn't feel broken, just unavailable. The skeleton teaches the UI layout before the user has access. Downside: skeletons are typically associated with loading states, which could cause confusion ("is this still loading?"). The dashed border on skeleton cards partially addresses this. Adds implementation complexity: need to render per-page skeleton variants. Best choice if spatial continuity and low interruption are priorities over explicit instruction.
