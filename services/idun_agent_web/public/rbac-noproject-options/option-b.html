Copy tradeoffs: The numbered steps add reading load but reduce cognitive uncertainty — users know exactly how many steps there are and what each one involves. The "who is the owner" passive box for non-owners answers the most common follow-up question before they ask it. Steps copy is deliberately brief; longer explanations belong in docs, not the UI.

Affordance tradeoffs: The split layout makes clear there are two distinct concepts (situation + resolution) rather than one vague message. Downside: adds layout complexity — the left/right split may feel heavyweight if the user just glances and clicks the CTA. The responsive stacked version (mobile) degrades cleanly. Best choice when the user might be confused about why they're seeing an empty state, not just what to click.
