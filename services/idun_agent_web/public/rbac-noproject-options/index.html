NoProjectState — Design Options

Three distinct directions for the blocked/empty state shown when a user lands on a project-scoped page (agents, prompts, memory, MCP, guardrails, observability, integrations, SSO) with no active project or no project membership.

What you're choosing

Two variants must be covered in each option:

1. No project selected — user has projects but hasn't picked one yet. CTA leads to the top-navbar project selector.

2. No accessible project — user has zero project memberships in this workspace. If workspace owner → "Create project" CTA to Settings › Projects. If non-owner → passive copy directing them to ask an owner.

The component renders inside the page's content area; the navbar, sidebar, and page frame remain visible around it.
