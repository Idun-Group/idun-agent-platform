Approach B — Dialog-Driven
All state changes go through confirmation dialogs. Role changes, adding members, leaving workspaces — each gets a focused modal explaining the consequences. Safer, more explicit. Better for teams where mistakes are costly.

Approach C — Command Palette + Contextual Actions
Workspace switching uses a command palette (⌘K). Role changes and member additions use contextual action menus. Feels native and keyboard-friendly. Scales well as features grow. Combines the speed of Approach A with the safety of Approach B.
