Verdict

Pros

  • Carries the most information per toast — title + description + timestamp
  • Timestamp is genuinely useful for multi-step workflows
  • Icon tile is visually distinctive without relying on background color
  • Scales gracefully: works with just a title when no description is provided
  • Aligns with existing notification paradigms (Slack, Figma, Linear)

Cons

  • Tallest of the three — takes more vertical space on mobile
  • Callers must provide both a title string and an optional description
  • Without a description, the two-row structure can feel sparse
  • API change required: notify.success(title, description?) instead of notify.success(message)

Best for

  • Power users who run multiple async operations in parallel
  • Error states that benefit from a short explanation inline
  • Teams that want to match a desktop notification UX pattern
  • Scenarios where timestamps matter (agent deploys, job completions)
