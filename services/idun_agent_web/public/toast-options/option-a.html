Verdict

Pros

  • Lowest visual weight of the three — never fights content
  • Only 2 visual layers (stripe + border) — simple to maintain
  • Responsive by construction — container is the constraint
  • X button is absolute-positioned, never clips at any width
  • No anchor-center, no !important, no hacks
  • Fits naturally with Linear / Raycast / Vercel aesthetics

Cons

  • Thin stripe is subtle — errors may not feel urgent enough
  • No progress bar — users get no sense of auto-dismiss timing
  • Monochrome background means users must read the text to know severity
  • Single message line only — no room for a description

Best for

  • Developer tools and admin dashboards where the UI is already dense
  • Users who find heavy toasts distracting
  • Situations where most toasts are success / info (low urgency)
  • Teams that want to implement the simplest possible system first
