Project Members — 3 Design Options

How should users navigate to manage members of a specific project?

Option A Dropdown selector at the top

The Project Members tab stays as its own settings page, but has a project picker dropdown at the top. Switching the dropdown loads that project's members without leaving the page.

Pros

Simple. Keeps a dedicated settings tab for project members. Quick switching.

Cons

Disconnected from the Projects page — two places to think about projects. Dropdown can feel redundant with the header project selector.
Option B Accordion on the Projects page

Remove the separate "Project Members" tab entirely. On the All Projects page, each project is an expandable accordion. Click to expand and manage members inline. Everything about a project lives in one place.

Pros

Single source of truth for projects. Members are in context — no navigation needed. Removes one settings tab. Shows member counts at a glance.

Cons

Page can get long with many projects expanded. Mixes project config (rename/delete) with member management in one view.
Option C Members button → Slide panel

On the All Projects page, each project row has a "Members" button showing the count. Clicking it opens a slide-over panel on the right. You manage members in the panel without leaving the page. Remove the separate "Project Members" tab.
