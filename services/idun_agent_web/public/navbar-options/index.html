Workspace / Project selector — 3 design directions

All three directions solve the same problem: make it obvious at a glance that the navbar shows the active workspace and project and that both are switchable — without needing a click. Click-to-popover interaction is unchanged.

Option A — Stacked label
Tiny uppercase tertiary-color type label ("WORKSPACE") stacked above the value + chevron. Breadcrumb-like. No extra horizontal space.
Pros:Very compact. Label is always visible. Works at 375 px. Purple label ties into brand.
Cons:Two-line button increases header height slightly; label is small at ~9px.
Best for:Dense, professional UIs where header real estate is constrained.

Option B — Inline prefix
Muted "Workspace ·" prefix before the name on one line. Reads like a sentence. Vercel / Linear aesthetic — explicit without visual noise.
Pros:Self-explanatory at a glance. Single-line keeps header clean. Familiar pattern.
Cons:Wider at long names. Prefix may feel redundant after users learn the UI.
Best for:First-time users; products with many workspaces where confusion is common.

Option C — Icon + pill
Building2 icon for workspace, FolderOpen for project, each wrapped in a subtle bordered pill. Icon-driven recognition — no text label needed.
Pros:Icon meaning is universal. Works well at mobile. Heavier visual weight signals interactivity.
Cons:Icons alone may not be enough for first-time users unfamiliar with the icon convention.
Best for:Returning users; icon-fluent audiences; mobile-first layouts.

Main tradeoffs at a glance

Density Option A packs the most into the smallest space. Option C is widest at desktop, narrowest at mobile (icon only).
Explicitness Option B is the most text-explicit. Option A is explicit but smaller. Option C relies on icon recognition.
Icon recognition Option C wins for returning users but requires icon literacy. Options A and B don't depend on icons.
Recommendation Option A if you want brand cohesion and compactness. Option B if first-time user clarity outweighs density. Option C if you plan to drop text labels at mobile in the future.
